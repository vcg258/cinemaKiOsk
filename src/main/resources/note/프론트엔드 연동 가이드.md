## 2. 컨트롤러 작성 규칙

### 2-1. Model 속성 키 이름 준수

각 HTML 템플릿 파일 **상단 주석**에 `Controller → View 전달 Model 속성` 항목이 명시되어 있습니다.
`model.addAttribute()` 작성 시 **주석에 적힌 키 이름을 반드시 그대로 사용**해 주세요.
키 이름이 다를 경우 Thymeleaf에서 데이터를 읽지 못합니다.

**예시 (`movie/detail.html` 기준)**

```java
// MainController.java
@GetMapping("/movie/{id}")
public String detail(@PathVariable Long id, Model model) {
    MovieDTO movie = movieService.findById(id);
    model.addAttribute("movie",     movie);                       // 키: "movie"
    model.addAttribute("isSoldOut", movieService.isSoldOut(id)); // 키: "isSoldOut"
    model.addAttribute("posterUrl", movie.getPosterUrl());        // 키: "posterUrl"
    return "movie/detail";
}
```

### 2-2. URL 매핑 기준

| 컨트롤러 | 담당 URL |
|---------|---------|
| `MainController` | `GET /movie/list`, `GET /movie/{id}` |
| `BookingController` | `GET /booking/schedule`, `GET /booking/seat` |
| `PaymentController` | `GET /payment/payment`, `GET /payment/result` |
| `MypageController` | `GET /mypage/lookup`, `GET /mypage/detail` |
| `AdminAuthController` | `POST /admin/login`, `GET /admin/dashboard` |
| `AdminController` | `GET /admin/stats/**`, `GET /admin/refund`, `POST /admin/refund/**`, `GET /admin/movie/**`, `GET /admin/theater/**` |

### 2-3. 레이아웃 구분

| 대상 | 사용 레이아웃 |
|------|------------|
| 고객용 전체 페이지 | `layout/base.html` |
| 관리자용 전체 페이지 | `layout/admin-base.html` |
| 관리자 로그인 페이지 | 레이아웃 없음 (단독 HTML) |

## 3. REST API 엔드포인트 명세

프론트엔드 JS(`fetch` / `axios`)가 호출하는 엔드포인트 목록입니다.
아래 형태에 맞춰 구현해 주세요. 응답은 해당 **DTO 기준 JSON**으로 반환해 주세요.
백엔드 담당자 분들 협의 후 확정 및 수정해 주시면 그 내용 토대로 작업하겠습니다.

### 고객 영역

| 기능 | Method | URL | 요청 파라미터 / Body | 응답 |
|------|--------|-----|-------------------|------|
| 영화 목록 조회 | GET | `/api/movies` | `?status=NOW\|UPCOMING&genre=&rating=&keyword=` (선택) | `List<MovieDTO>` |
| 영화 상세 조회 | GET | `/api/movies/{movieId}` | - | `MovieDTO` |
| 상영 스케줄 조회 | GET | `/api/schedules` | `?movieId=` | `List<ScheduleDTO>` |
| 좌석 현황 조회 | GET | `/api/schedules/{scheduleId}/seats` | - | `List<SeatDTO>` |
| 예매 임시 점유 | POST | `/api/reservation/hold` | `{ scheduleId, seatNumbers[], adultCount, teenCount }` | `ReservationDetailsDTO` |
| 포인트 인증 | POST | `/api/auth/phone` | `{ phone }` | `{ memberId, phone, point }` |
| 포인트 잔액 조회 | GET | `/api/members/point` | `?phone=` | `{ point }` |
| 결제 요청 | POST | `/api/payment` | `{ reservationId, discountPolicyId, usePoint, paymentMethod }` | `{ reservationId }` |
| 예매 조회 | GET | `/api/mypage/lookup` | `?reservationId=` | `ReservationDetailsDTO` |
| 예매 취소 | DELETE | `/api/reservation/{id}` | - | - |

### 관리자 영역

| 기능 | Method | URL | 요청 파라미터 / Body | 응답 |
|------|--------|-----|-------------------|------|
| 환불 조회 | GET | `/api/admin/refund` | `?reservationId=` 또는 `?phone=` | `ReservationDetailsDTO` |
| 환불 처리 | POST | `/api/admin/refund/{reservationId}` | - | - |
| 영화 등록 | POST | `/api/admin/movie` | `MovieDTO` | `MovieDTO` |
| 영화 수정 | PUT | `/api/admin/movie/{movieId}` | `MovieDTO` | `MovieDTO` |
| 상영 중지 | PATCH | `/api/admin/movie/{movieId}/stop` | - | - |
| 상영관 수정 | PUT | `/api/admin/theater/{no}` | `TheaterDTO` | `TheaterDTO` |
| 통계 조회 | GET | `/api/admin/stats/{type}` | `?startDate=&endDate=` | `List<StatisticsDTO>` |

> `{type}` : `daily` / `monthly` / `by-day` / `by-hour` / `by-movie`

## 4. WebSocket 연동 (UC-03 좌석 선택)

`booking/seat.html` 화면에서 실시간 좌석 동기화에 사용합니다.
`WebSocketController` 구현 시 아래 경로 참고해 주세요.

```
연결 엔드포인트 : /ws  (SockJS)
구독 경로       : /topic/seat/{scheduleId}   ← 서버 → 클라이언트 브로드캐스트
발행 경로       : /app/seat/select           ← 클라이언트 좌석 선택 시 전송
                  /app/seat/release          ← 클라이언트 이탈 / 해제 시 전송
```

메시지 형태는 `WebSocketController` 담당자(혜윰님)와 별도 협의 후 확정할 것.

### th:if="${false}" 제거 완료 여부

과거 JS 파일 없을 때 임시로 적용했던 `th:if="${false}"` 처리는 **완료된 파일에서 모두 제거**되었습니다.
stub 파일도 빈 파일로 생성해 두었으므로 404 에러는 발생하지 않습니다.

## 7. 주의사항

- 백엔드 로직(Service, Repository)은 프론트엔드 파일에 구현하지 않습니다.
- `th:inline="javascript"` 없이 Thymeleaf 변수를 JS에서 직접 참조하는 코드는 작성하지 말아 주세요.
- `payment/payment.html`의 `th:inline="javascript"` 데이터 브릿지 변수 목록:
  `RESERVATION_ID`, `SCHEDULE_ID`, `SEAT_NUMBERS`, `SEAT_COST`, `ADULT_COUNT`, `TEEN_COUNT`, `DISCOUNT_POLICIES`, `BONUS_POLICIES`, `MEMBER_PHONE`, `MEMBER_POINT`
- `booking/seat.html`의 WebSocket 연결은 `connectWS()` / `disconnectWS()` 함수로 분리되어 있습니다.
  연결 실패 시 폴링 fallback 처리가 포함되어 있으니 참고해 주세요.

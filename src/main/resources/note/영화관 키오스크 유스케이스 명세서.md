**영화관 키오스크 유스케이스 명세서**

통합본

**Actor 구분**

| Actor 구분 | Actor | 설명 |
| :---- | :---- | :---- |
| **Primary Actor** | 회원 (고객) | 키오스크를 통해 영화를 조회·예매·결제하고 포인트를 사용하는 서비스 이용자 |
| **Secondary Actor** | 관리자 | 시스템 내 영화·상영관·통계를 관리하고 환불을 처리하는 운영 주체 |

**회원 (고객) Use Cases  —  UC-01 \~ UC-10**

| Use Case 이름 | UC-01  상영작 목록 조회 |
| :---- | :---- |
| **목표 Goal** | 회원이 현재 상영 중인 영화 목록을 장르·관람등급·상영관 유형 필터를 활용해 조회한다. |
| **행위자 Actor** | 회원 (고객) |
| **사전 조건Precondition** | · 키오스크(또는 웹) 메인 화면에 접근한 상태이다. · 상영 중인 영화가 1편 이상 등록되어 있다. |
| **후조건Postcondition** | · 조건에 맞는 상영작 목록이 화면에 표시된다. · 회원은 원하는 영화를 선택해 상세 조회로 이동할 수 있다. |
| **기본 흐름Main Flow** | 1\. 회원이 메인 화면에서 '상영작 목록' 메뉴를 선택한다. 2\. 시스템이 현재 상영 중인 전체 영화 목록(포스터·제목·장르·등급·상영관 유형)을 조회하여 표시한다. 3\. 회원이 필터(장르 / 관람등급 / 상영관 유형)를 선택하거나 검색어를 입력한다. 4\. 시스템이 선택된 조건에 맞는 영화 목록을 재조회하여 표시한다. 5\. 회원이 원하는 영화를 선택하면 UC-02(상영작 상세 조회)로 이동한다. |
| **대체 흐름Alternative Flow** | 1\. 회원이 입력한 검색어나 필터 조건에 맞는 영화가 없을 경우 → "해당 조건의 상영작이 없습니다"라는 메시지를 표시하고 전체 목록으로 돌아가기 버튼을 제공한다. 2\. 상영 예정작 조회 → 예매는 불가능하지만 정보 조회는 가능한 개봉 예정 영화 목록을 표시한다. 3\. '필터 초기화' 버튼을 누르면 초기 전체 목록 상태로 돌아간다. |

| Use Case 이름 | UC-02  상영작 상세 조회 |
| :---- | :---- |
| **목표 Goal** | 회원이 특정 영화의 상세 정보(줄거리·러닝타임·감독/배우·관람등급·상영관 유형)를 확인한다. |
| **행위자 Actor** | 회원 (고객) |
| **사전 조건Precondition** | · UC-01에서 상영작 목록이 표시된 상태이다. · 조회할 영화가 시스템에 등록되어 있다. |
| **후조건Postcondition** | · 선택한 영화의 상세 정보가 화면에 표시된다. · 회원은 해당 영화의 예매로 이동할 수 있다. |
| **기본 흐름Main Flow** | 1\. 회원이 목록에서 영화를 선택한다. 2\. 시스템이 해당 영화의 상세 정보(포스터·줄거리·러닝타임·감독·배우·관람등급·상영관 유형)를 표시한다. 3\. 회원이 '예매하기' 버튼을 선택하면 UC-03(예매하기)로 이동한다. |
| **대체 흐름Alternative Flow** | 선택한 영화가 당일 상영이 모두 종료되었거나 전석 매진된 경우 → '예매하기' 버튼을 비활성화하거나 '매진' 안내 문구를 표시한다. |

| Use Case 이름 | UC-03  예매하기 |
| :---- | :---- |
| **목표 Goal** | 회원이 원하는 영화·상영 시간대·좌석을 선택하여 결제 단계로 이동한다. |
| **행위자 Actor** | 회원 (고객) |
| **사전 조건Precondition** | · 상영작 상세 화면 또는 예매 화면에 접근한 상태이다. · 해당 영화의 잔여 좌석이 1석 이상 존재한다. · 영화 시작 5분 후까지이어야 한다. |
| **후조건Postcondition** | · 결제 화면(UC-04)으로 이동한다. |
| **기본 흐름Main Flow** | 1\. 회원이 영화 상세 화면에서 '예매하기' 버튼을 선택한다. 2\. 시스템이 해당 영화의 상영 날짜 목록을 표시한다. 3\. 회원이 날짜를 선택하면 해당 날짜의 상영 시간대 목록이 표시된다. 4\. 회원이 상영 시간대를 선택한다. 5\. 회원이 인원 수(성인/청소년)를 입력한다. 6\. 좌석 목록이 표시되며, 원하는 좌석을 선택한다. 단 여러 명의 좌석 선택 시 바로 옆으로 자동 선택. |
| **대체 흐름Alternative Flow** | 1\. 기본 흐름 도중 영화 시작 후 5분이 지난다면 메시지 표시 후 UC-02(상영작 상세 조회)로 돌아감. 2\. 다른 회원이 동시에 같은 좌석 선택 시 (웹소켓) \- 먼저 선택한 회원: 해당 좌석이 "선택 중" 상태로 표시됨. \- 나중에 선택한 회원: "이미 다른 고객이 선택 중인 좌석입니다" 메시지 표시, 해당 좌석 선택 불가. 3\. 선택 중이던 회원이 결제를 완료한 경우 (웹소켓) \- 대기 중인 회원 화면에서 해당 좌석이 즉시 '예매 완료'로 전환. \- "선택하신 좌석이 예매 완료되었습니다. 다른 좌석을 선택해 주세요" 메시지 표시. 4\. 선택 중이던 회원이 웹소켓 연결을 종료(이탈)한 경우 \- 즉시 해당 좌석 임시 점유 해제, "선택 가능" 상태로 전환. |

| Use Case 이름 | UC-04  결제 처리 |
| :---- | :---- |
| **목표 Goal** | 회원이 예매 정보를 최종 확인하고 결제 수단을 선택하여 결제를 완료한다. |
| **행위자 Actor** | 회원 (고객) |
| **사전 조건Precondition** | · UC-03(예매하기)에서 좌석 선택이 완료된 상태이다. · 선택 좌석이 임시 점유 상태이다. |
| **후조건Postcondition** | · 결제가 승인되면 예매 상태가 '결제 완료'로 변경된다. · 잔여 좌석 수가 예매 인원만큼 감소한다. · 포인트 사용 시 회원 포인트가 차감된다. · 결제 금액의 일정 비율이 포인트로 적립된다. (UC-07) · 영수증(예매 확인증)이 화면에 표시된다. |
| **기본 흐름Main Flow** | 1\. 시스템이 최종 결제 확인 화면(영화·날짜·시간·좌석·총 결제 금액)을 표시한다. 2\. 여러 할인(이벤트, 쿠폰, 관람권 등)을 선택해 적용한다. 3\. 회원이 포인트 사용 여부를 선택한다. (UC-05 / UC-06 연계) 4\. 회원이 결제 수단(신용카드·현금·간편결제 등)을 선택한다. 5\. 회원이 '결제하기' 버튼을 선택한다. 6\. 시스템이 결제 모듈에 승인 요청을 전송한다. 7\. 결제 모듈이 승인 응답을 반환한다. 8\. 시스템이 예매 상태를 '결제 완료'로 변경하고 잔여 좌석을 차감한다. 9\. 시스템이 결제 금액 기준으로 포인트를 자동 적립한다. (UC-07) 10\. 티켓과 영수증이 실물 / 혹은 메시지 등으로 출력된다. |
| **대체 흐름Alternative Flow** | 결제 승인이 실패한 경우: → 시스템이 '결제에 실패하였습니다. 다시 시도해 주세요.' 메시지를 표시한다. → 임시 점유 좌석을 유지하고 결제 수단 선택 화면으로 돌아간다. 임시 점유 시간(예: 10분)이 초과된 경우: → 시스템이 좌석 점유를 해제하고 '시간 초과로 예매가 취소되었습니다.' 메시지를 표시한다. → UC-03(예매하기)부터 재시작한다. |

| Use Case 이름 | UC-05  휴대폰 인증 |
| :---- | :---- |
| **목표 Goal** | 회원이 휴대폰 번호를 입력하여 본인 인증을 완료한다. |
| **행위자 Actor** | 회원 (고객) |
| **사전 조건Precondition** | UC-04 결제 진행 중 포인트 사용 절차에 진입한다. |
| **후조건Postcondition** | · 휴대폰 인증이 완료되어 이후엔 인증할 필요가 없어진다. · 포인트 조회와 사용이 가능하다. |
| **기본 흐름Main Flow** | 1\. UC-04 결제 처리 도중 포인트 사용 시 휴대폰 인증이 되어 있지 않다면 인증창을 표시한다. 2\. 회원이 휴대폰 번호를 입력한다. 3\. 시스템이 입력된 번호로 회원 정보를 조회하여 인증을 완료한다. 4\. 고객은 포인트를 조회할 수 있으며, 사용 여부를 선택한다. |
| **대체 흐름Alternative Flow** | 1\. 시스템이 입력된 휴대폰 번호와 일치하는 회원 정보가 없음을 감지하면 → '등록되지 않은 번호입니다.' 메시지를 표시한다. → 회원이 번호를 다시 입력하거나 회원 가입 화면으로 이동할 수 있다. 2\. 회원이 숫자 외 문자 포함, 자릿수 부족 등 유효하지 않은 형식으로 번호를 입력하면 → '올바른 휴대폰 번호 형식으로 입력해 주세요.' 메시지를 표시한다. |

| Use Case 이름 | UC-06  포인트 사용 |
| :---- | :---- |
| **목표 Goal** | 회원이 결제 시 보유 포인트를 차감하여 할인 혜택을 받는다. |
| **행위자 Actor** | 회원 (고객) |
| **사전 조건Precondition** | · UC-04 결제 처리가 진행 중인 상태이다. · 선택 좌석이 임시 점유 상태이다. · 회원의 포인트 잔액이 최소 사용 단위 이상이다. · UC-05 휴대폰 본인 인증을 마친 상태이다. |
| **후조건Postcondition** | · 사용된 포인트만큼 회원 포인트 잔액이 차감된다. · 최종 결제 금액이 사용 포인트만큼 감소된다. · 포인트 사용 내역이 시스템에 기록된다. |
| **기본 흐름Main Flow** | 1\. 결제 화면에서 시스템이 현재 포인트 잔액을 표시한다. 2\. 회원이 사용할 포인트 금액을 입력하거나 '전액 사용' 버튼을 선택한다. 3\. 시스템이 포인트 사용액만큼 결제 금액에서 차감하여 최종 결제 금액을 표시한다. 4\. 회원이 결제를 진행하면 UC-04(결제 처리) 기본 흐름을 따른다. 5\. 결제 완료 후 시스템이 포인트 잔액을 차감하고 사용 내역을 기록한다. |
| **대체 흐름Alternative Flow** | 포인트가 요금보다 높을 때 고객이 '전액사용'을 선택하면 → 결제 금액을 0원으로 처리하고, 차감 포인트는 결제 금액 이내로 조정된다. → 이후 UC-04 결제 처리의 결제 수단 선택 절차가 사라진다. |

| Use Case 이름 | UC-07  포인트 적립 |
| :---- | :---- |
| **목표 Goal** | 결제 완료 시 결제 금액의 일정 비율을 포인트로 자동 적립한다. |
| **행위자 Actor** | 회원 (고객) |
| **사전 조건Precondition** | · UC-04(결제 처리)가 성공적으로 완료된 상태이다. · 회원이 포인트 적립 대상(휴대폰 번호 확인 완료)이다. |
| **후조건Postcondition** | · 결제 금액 기반 포인트가 회원 계정에 적립된다. · 포인트 적립 내역이 시스템에 기록된다. |
| **기본 흐름Main Flow** | 1\. 시스템이 결제 완료 후 적립 정책(결제 금액의 n%)에 따라 적립 포인트를 산출한다. 2\. 시스템이 회원의 포인트 잔액에 산출된 포인트를 가산한다. 3\. 시스템이 적립 내역(날짜·예매 번호·적립 포인트·누적 포인트)을 저장한다. 4\. 결제 확인증 화면에 '○○ 포인트 적립 완료' 메시지가 표시된다. |
| **대체 흐름Alternative Flow** | 입력된 휴대폰 번호로 회원 정보를 조회하였으나 일치하는 계정이 없다면 → 결제 확인증에 '포인트 적립 불가 \-- 회원 정보를 확인해 주세요' 메시지가 표시된다. |

| Use Case 이름 | UC-08  예매 조회 |
| :---- | :---- |
| **목표 Goal** | 고객이 본인의 예매 내역을 조회하여 티켓 출력 또는 취소를 진행한다. |
| **행위자 Actor** | 고객 |
| **사전 조건Precondition** | 예매에 대한 기록이 DB에 저장되어 있다. |
| **후조건Postcondition** | · 선택한 예매 정보가 화면에 표시된다.   · 출력 선택 시 티켓이 인쇄되고 예매가 확정된다. · 취소 선택 시 환불이 자동으로 처리된다. |
| **기본 흐름Main Flow** | 1\. 화면에서 '조회'를 클릭한다. 2\. 예매번호를 입력한다. (개인정보 보호로 인해 휴대폰 번호 조회 방식에서 변경) 3\. 해당 예매 정보를 보여준다. 4\. 출력 / 취소에 대한 선택지를 보여준다. 5-1. 출력을 선택한 경우 티켓이 인쇄되고 예매가 확정된다. 5-2. 취소를 선택한 경우 자동으로 환불이 진행된다. |
| **대체 흐름Alternative Flow** | 2\. 입력한 예매번호에 해당하는 예매 내역이 없다면 '예매 내역을 찾을 수 없습니다.' 메시지 표기 후 5초 뒤 메인 화면으로 이동. |

**관리자 Use Cases  —  UC-11 \~ UC-21**

| Use Case 이름 | UC-11  관리자 로그인 |
| :---- | :---- |
| **목표 Goal** | 관리자 기능의 해금 |
| **행위자 Actor** | 관리자 |

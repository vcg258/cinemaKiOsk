TS Websocket
send(data) : data를 연결된 서버로 보낸다
close(code, reason) : 연결 종료
Onopen : 연결이 성공적으로 개설되었을 때 호출
Onmessage : 서버로부터 메시지를 수신했을 때 호출
Onerror : 연결중 에러가 발생했을 때 호출
Onclose : 연결이 닫혔을 때 호출



Java TextWebSocket
afterConnectionEstablished: 연결 성공 시 실행.
handleTextMessage: 메시지 수신 시 실행 (가장 핵심).
afterConnectionClosed: 연결 종료 시 실행.
handleTransportError: 통신 에러 발생 시 실행.



handleMessage 이 메서드는 모든 메시지(텍스트, 바이너리, 제어 메시지)가 거쳐 가는 최상위 관문입니다. 기능: 수신된 메시지의 타입을 확인하고, handleTextMessage나 handleBinaryMessage로 배분하는 역할을 합니다. 사용 용도: 만약 메시지 타입에 상관없이 공통적으로 로깅을 남기거나, 특정 조건에서 메시지 처리를 차단해야 할 때 이 메서드를 오버라이드하여 로직을 추가할 수 있습니다.

handleBinaryMessage TextWebSocketHandler를 상속받더라도 바이너리 데이터(이미지, 파일 등)가 들어올 수 있습니다. 기능: 바이너리 메시지를 처리합니다. 사용 용도: 기본적으로 TextWebSocketHandler는 이 메서드를 만났을 때 에러를 내거나 무시하지만, 필요에 따라 텍스트와 파일을 동시에 주고받는 환경을 만들 때 오버라이드하여 구현할 수 있습니다.

supportsPartialMessages 이 메서드는 불리언(boolean) 값을 반환합니다. 기능: 매우 큰 메시지가 여러 번에 걸쳐 나누어 들어올 때(Partial Message), 이를 허용할지 결정합니다. 설정: * false (기본값): 전체 메시지가 다 도착할 때까지 버퍼에 쌓았다가 한 번에 전달합니다. true: 메시지 조각이 올 때마다 메서드를 실행합니다. 이때는 message.isLast() 메서드를 통해 마지막 조각인지 확인하는 로직이 추가로 필요합니다.

## API 호출 URL (백엔드 Spring Boot 기준)

> 프론트에서 axios 등으로 호출하는 URL. Vite 프록시가 `/api` → `http://localhost:8080/api` 로 포워딩.
> Spring Boot 컨트롤러는 `@RequestMapping("/api/...")` 형태로 선언.

### 고객 영역
| HTTP Method | API URL | 설명 |
|-------------|---------|------|
| GET | `/api/movie/list` | 영화 목록 조회 |
| POST | `/api/movie/detail` | 영화 상세 조회 |
| POST | `/api/booking/schedule` | 상영 스케줄 선택 |
| POST | `/api/booking/seat` | 좌석 선택 화면 |
| POST | `/api/payment/payment` | 결제 처리 |
| GET | `/api/payment/result` | 결제 결과 조회 |
| WS | `/ws/seats/{scheduleId}` | 좌석 실시간 동기화 (WebSocket — `/api` 없음) |

### 관리자 영역
| HTTP Method | API URL | 설명 |
|-------------|---------|------|
| GET / POST | `/api/admin/login/main` | 로그인 화면·처리 |
| GET | `/api/admin/statistics/dashboard` | 통계 대시보드 |
| GET | `/api/admin/statistics/stats/daily` | 일일 통계 |
| GET | `/api/admin/statistics/stats/monthly` | 월별 통계 |
| GET | `/api/admin/statistics/stats/by-day` | 요일별 통계 |
| GET | `/api/admin/statistics/stats/by-hour` | 시간대별 통계 |
| GET | `/api/admin/statistics/stats/by-movie` | 영화별 통계 |
| GET / POST | `/api/admin/management/movie/form` | 영화 등록·수정 |
| GET | `/api/admin/management/movie/list` | 영화 목록 (관리) |
| GET / POST | `/api/admin/management/movie/manage` | 영화 상태 변경 |
| GET | `/api/admin/management/theater/list` | 상영관 목록 |
| GET / POST | `/api/admin/management/theater/edit` | 상영관 수정 |
| GET | `/api/admin/management/seat/list` | 좌석 목록 |
| GET / POST | `/api/admin/management/seat/edit` | 좌석 수정 |
| GET | `/api/admin/management/policy/list` | 정책 목록 |

| GET / POST | `/api/admin/management/policy/manage` | 정책 활성화 관리 |
| GET | `/api/admin/refund/` | 환불 조회 |
| POST | `/api/admin/refund/{reservationId}` | 환불 처리 |

## 개발 환경

- **Node**: 22.22.0
- **Vite 개발 서버**: `http://localhost:3000`
- **백엔드**: `http://localhost:8080` (Spring Boot)
- **프록시 경로**: `/api`, `/ws` → 8080 자동 프록시
- **빌드 명령**: `npm run dev` (개발) / `npm run build` (프로덕션)

## 참고사항

- **화면 기준**: 고객용 페이지는 FHD 세로형 `1080 × 1920` 기준
- **다크모드**: 관리자 영역만 적용 (`localStorage` `adminTheme` 키)
- **WebSocket**: `/ws/seats/{scheduleId}` — STOMP over SockJS, 좌석 실시간 동기화. `/api` 접두사 없이 별도 프록시 경로 사용 (TODO: `useWebSocket.ts` 구현 예정)
- **인증**: `AuthContext` + `PrivateRoute` 조합으로 관리자 페이지 보호
- **목업 데이터**: `src/api/mockData.ts` — 백엔드 API 완성 전까지 사용
